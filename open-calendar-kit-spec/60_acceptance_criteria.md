# Akzeptanzkriterien

OpenCalendarKit Public 1.0 ist fachlich erreicht, wenn:

1. Das Plugin als eigenständiges Plugin unter `open-calendar-kit` lauffähig ist.
2. Das Plugin keine Reservierungsfunktion mehr enthält.
3. Alle sichtbaren Texte internationalisierbar sind.
4. Öffnungszeiten im Backend gepflegt und im Frontend ausgegeben werden können.
5. Der Monatskalender funktioniert.
6. Schließtage im Kalender berücksichtigt werden.
7. Die Statusanzeige global aktivierbar/deaktivierbar ist.
8. Die Event Notice per Shortcode funktioniert.
9. Aktivierung/Reaktivierung keine bestehenden Daten überschreibt.
10. Uninstall keine Benutzerdaten ungefragt löscht.
11. Das Plugin mobil und auf Desktop nutzbar ist.
12. Grunddokumentation für GitHub vorhanden ist.
13. Ein WordPress.org-nahes `readme.txt` ist vorhanden.
14. Lizenzangaben sind zwischen Plugin-Header, `readme.txt`, `README.md` und `LICENSE` konsistent.
15. Aus dem Verzeichnis `open-calendar-kit` lässt sich ein installierbares Plugin-ZIP ohne Entwicklungsreste ableiten.
